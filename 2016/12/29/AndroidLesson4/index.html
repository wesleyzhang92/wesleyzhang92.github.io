四大组件之--活动

Posted on December 29, 2016

活动（Activity）是一种可以包含用户界面的组件，主要用于和用户进行交互。一个程序中可以包含零个或多个活动。

<!—more–>

活动的概念

实现的每一个活动都是Activity的子类，通常情况下每一个Activity都实现了不同的功能，不过每一个有多个活动的应用都应该指名一个作为main activity，即用户打开应用所见的活动，Activity之间可以互相启动。不同应用之间的活动也可以互相启动，在使用活动时，需要在应用的manifest中进行注册，并处理好Activity的生命周期。

配置Manifest文件

为了让应用可以使用创建的活动，就需要在manifest中对活动以及活动的一些属性进行声明。

声明活动

在manifest文件中添加\标签，必要的属性是指定Activity的name，对应于Activity的类名，更多的属性可以参看activity

管理活动的生命周期

在生命周期中一共包含了六个回调函数，分别为onCreate(), onStart(), onResume(), onPause(), onStop(), 和 onDestroy()，通常不需要实现所有的回调函数。

回调函数

  1. onCreate()

    必须实现onCreate函数，当系统首次创建这个活动时执行，整个生命周期执行一次。接受参数 savedInstanceState, 是一个保存了活动之前的状态的 Bundle 对象，如果活动之前从没出现，则值为null。

  2. onStart()

    这个阶段使活动变得用户可见，执行的速度很快，马上就进入到resume的状态。

  3. onResume()

    Activity进入到前台，开始和用户交互，应用将一直保持这种状态直到一些中断事件发生，状态会进入到paused，当从pause阶段返回时，会再次调用onResume()。 You should implement onResume() to initialize components that you release during onPause(), and perform any other initializations that must occur each time the activity enters the Resumed state. For example, you should begin animations and initialize components that the activity only uses when it has user focus.

  4. onPause()

    当用户离开当前活动时会调用这个方法，执行时间很短，不能在这个阶段保存当前活动的数据。You can use the onPause() method to release system resources, such as broadcast receivers, handles to sensors (like GPS), or any resources that may affect battery life while your activity is paused and the user does not need them.onPause阶段结束之后，Activity仍处在pause的状态，直到重新调用resume或者彻底对用户不可见。

  5. onStop()

    当Activity对用户彻底不可见时，就进入到了Stopped状态。在onStop方法中，应用会释放几乎所有用户不在使用的资源，可以防止内存泄漏。也可以利用onStop()进行一些需要消耗CPU的操作，如将数据保存到数据库等。在Stopped状态，Activity仍然驻留内存，所有View对象的状态都被系统保留，即使由于内存不足被系统kill，UI控件的状态信息仍然以bundle形式保存。

  6. onDestory()

    finish()的调用或者系统内存不足会导致进入onDestory，当屏幕方向转变时，也会先Destroy然后重新创建一个活动。
